## 📦 What Data We Collect

Miko collects **only the minimum data** required to provide its features. We believe in minimal data collection — if a feature does not need your data, we do not collect it.

<div align="center">

| Data Type | Why We Collect It | Stored Permanently? |
|:---|:---|:---:|
| **Discord User IDs** | To identify users across commands (warns, AFK, reminders) | ✅ Yes |
| **Discord Server (Guild) IDs** | Per-server configuration: prefix, autorole, logs, sticky | ✅ Yes |
| **Discord Channel IDs** | Log channels, slowmode targets, nuke protection | ✅ Yes |
| **Discord Role IDs** | Autorole setup and moderation role tracking | ✅ Yes |
| **Message Content** | Snipe command & AFK detection only — not stored after bot restart | ❌ Temporary |
| **IGN / Tier Data** | Ranking features submitted via `/rp` command | ✅ Firebase |
| **Warn / Mod Records** | Warnings issued via `/warn` or `m!warn` per user per server | ✅ Yes |
| **AFK Status & Message** | Set via `/afk` — cleared when user returns or bot restarts | ❌ Temporary |

</div>

### What We Do NOT Collect

```
✗  Passwords or login credentials
✗  Payment or financial information
✗  Real names or physical addresses
✗  Email addresses
✗  IP addresses
✗  Private messages between users
✗  Voice chat audio or transcripts
✗  Any data from private DMs (except DMs Miko sends via commands like /dm)
```

## ⚙️ How We Use Your Data

Every piece of data collected has a specific, limited purpose:

- **User IDs** → Power commands like `/warn`, `/afk`, `/remind`, `/userinfo`, and game leaderboards
- **Server IDs** → Store per-server settings: custom prefix, autorole, log channel, sticky messages
- **Channel & Role IDs** → Enable features like `/lock`, `/setlogs`, `/autorole`, and `/slowmode`
- **Message Content** → Power the `/snipe` command (last deleted message) and AFK auto-detection — lives only in RAM, cleared on bot restart
- **IGN / Tier Data** → Stored in Firebase Realtime Database to power ranking systems
- **Warn Records** → Allow moderators to track, view, and clear member warnings per server
- **Join/Leave events** → Send notifications to your configured log channel via Discord webhooks

> **We never use your data for advertising, profiling, or any purpose beyond running Miko's features.**

## 🔒 Data Storage & Security

<div align="center">

| Storage Type | What Is Stored There | Who Can Access |
|:---|:---|:---|
| **Firebase Realtime Database** | IGN / Tier ranking data | Bot owner & authorized devs only |
| **Local Server Config** | Server prefix, autorole, log channel, sticky messages | Bot owner & authorized devs only |
| **RAM (Temporary)** | Snipe cache, AFK status | Cleared on every bot restart |

</div>

**Security measures in place:**

- 🔐 Bot token stored in `.env` — never hardcoded or exposed
- 🔐 Firebase credentials kept private on the hosting server
- 🔐 Webhook URLs stored in config, never exposed to users
- 🔐 Hosting via Orihost / Pterodactyl panel with restricted access
- 🔐 Data accessible only to bot owner (`levi_ackerman.09`) and authorized developers
- 🔐 No third-party analytics or tracking scripts used anywhere

## 🗑️ Data Deletion

You have the right to request deletion of your data at any time.

**Automatic deletion:**
- Temporary data (AFK status, snipe cache) is cleared every time the bot restarts
- Removing Miko from your server stops all new data collection for that server immediately

**Manual deletion — what you can request:**
- All warning records for a user in a specific server
- IGN / Tier data linked to your User ID
- Your server's full configuration data
- Any stored reminders or AFK entries

**To request data deletion**, contact the developer (see [Contact](#-contact) below). We will process your request within **48 hours**.

## 👶 Users Under 13

Miko is a Discord bot and operates fully under **Discord's Terms of Service**, which requires all users to be **at least 13 years old** (or the minimum age in their country).

- We do not knowingly collect data from users under 13
- If we become aware that a user under 13 has submitted data (e.g. via `/rp`), we will delete it immediately
- Parents or guardians may contact us to request data removal for a minor
